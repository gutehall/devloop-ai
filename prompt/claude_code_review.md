## What to review

## Review checklist
- Correctness, edge cases, error handling
- Security & privacy
- Performance / complexity
- API design & backwards compatibility
- Tests and observability (logs/metrics/tracing)
- Readability and maintainability
- Documentation and release notes

## Output format
1) Summary verdict (Approve / Request changes / Comment)
2) Major issues (must-fix) with evidence and suggested patches
3) Minor issues (nits / optional)
4) Suggested follow-ups (tech debt)
