# Cursor PR Finalize Mode

Purpose:
Polish PR description and commit message for reviewers. Explain changes in plain English.

PR description should include:
- Summary (2–4 sentences)
- What changed (bullet list)
- Why (short rationale)
- What to check (areas reviewers should focus on)
- Assumptions made (document for reviewers)
- Manual test steps (if applicable)
- Testing & Validation
- Linked issues (Closes LIN-XXX)
- Known limitations & risks

Also produce a concise commit message. Keep language clear and jargon-free.
