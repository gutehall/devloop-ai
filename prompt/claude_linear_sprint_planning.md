# Linear — sprint plan from backlog

## Inputs
- Sprint length: <1w|2w|...>
- Team capacity: <people> / <points> / <hours>
- Current priorities: <priorities>
- Backlog (paste list): 
<backlog>

## Task
1. Propose a sprint goal.
2. Select a set of issues that fit capacity.
3. Identify dependencies, sequencing, and owners.
4. Flag risks and propose mitigations.
5. Provide a day-by-day / milestone plan.

## Output
- Sprint goal
- Committed issues (ordered)
- Stretch issues
- Dependency map
- Risk list
